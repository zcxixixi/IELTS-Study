# Strategy Notes

## Core Principle

IELTS Reading should be solved by evidence, not by feeling.

## Current Notes

### 0. Vocabulary should be reviewed inside the lesson, not only in a separate word list

For this study system:

- keep one main `wordbook.md` for active memorisation
- aim for `8-12` new items per day, with `10` as the default
- during a normal reading session, briefly review `3-5` older items before or during explanation
- then add only `5-10` new words that are truly reusable

Key idea:

`wordbook is for memory`
`lesson is for activation`

### 1. Read the question before reading the whole passage

Focus first on:

- names
- dates
- numbers
- comparison words
- absolute words

These usually help narrow the evidence area faster.

### 2. Treat synonym replacement as the default

The passage often does not repeat the question wording directly.
Look for:

- synonyms
- paraphrases
- cause-effect rewrites
- passive-active rewrites

### 3. Wrong options often feel partly true

Common traps:

- only one word differs
- statement matches the passage but not the question
- extreme wording
- wrong subject or wrong scope

### 4. Use structure to narrow the evidence zone first

Do not jump from keyword straight to answer.

Better sequence:

1. extract the keyword in the question
2. ask what function that idea plays in the passage structure
3. predict which paragraph type should contain it
4. locate the exact sentence inside that paragraph
5. only then match the option ending

Typical paragraph functions:

- background
- current progress
- limitation
- requirement
- social implication
- author judgement

Useful reminder:

`Keyword finds the topic.`
`Structure finds the paragraph.`
`Logic finds the answer.`

### 5. Use paragraph structure and core sentences to locate first, then use detail sentences to answer

This is a high-value method for matching-information style questions.

Do not assume every answer sits directly inside the paragraph core sentence.

Better sequence:

1. identify the information type in the question
2. map that information type to the passage structure
3. use the paragraph core idea to lock the likely paragraph
4. scan the nearby detail sentences for the exact answer
5. confirm the wording match or paraphrase

Key distinction:

- passage structure narrows the search zone
- paragraph core sentence identifies the paragraph function
- local detail sentence often contains the final answer

Useful reminder:

`Structure gets you there.`
`Detail gets you the mark.`

Typical information-type to paragraph-function mapping:

- value factors / definition -> definition paragraph
- historical mistake / negative consequences -> problem paragraph
- energy use / practical application -> use or market paragraph
- pests / diseases / outbreaks -> threat-management paragraph
- alternative label / example / list item -> support-detail sentence near the core idea

### Worked Example: Passage on tree value, low-use wood, and forest management

Passage skeleton:

- A: tree value depends on many factors
- B: past high-grading created low-use wood and forest problems
- C: low-use wood has practical energy-market uses
- D: TSI Cut as one management approach
- E: Salvage Cut for pest and disease response
- F: Shelterwood Cut for regeneration structure
- G: some low-use wood must still be retained

Question set:

- 14 bad outcomes for a forest when people focus only on its financial reward
- 15 reference to the aspects of any tree that contribute to its worth
- 16 mention of the potential use of wood to help run vehicles
- 17 examples of insects that attack trees
- 18 an alternative name for trees that produce low-use wood

How to solve:

#### 14 -> B

Question function:

- asks for negative consequences of a profit-only mindset

Structure reasoning:

- this belongs to the historical problem paragraph

Evidence pattern:

- B explains that landowners focused only on high-quality timber
- B then lists harmful results: lower value, competition, blocked regeneration, weaker forest health

Takeaway:

- when a question asks about harmful results from an old practice, go to the problem paragraph first

#### 15 -> A

Question function:

- asks for factors that determine value

Structure reasoning:

- this belongs to the opening definition paragraph

Evidence pattern:

- A lists species, size, form, condition, quality, function, and accessibility

Takeaway:

- when a question asks what contributes to value or worth, check the paragraph that defines the topic

#### 16 -> C

Question function:

- asks about practical energy use linked to vehicles

Structure reasoning:

- this belongs to the market/use paragraph

Evidence pattern:

- C says wood can be made into liquid fuels like ethanol and gasoline for lorries and cars

Takeaway:

- when a question asks how something can be used in practice, go to the application paragraph rather than the methods paragraphs

#### 17 -> E

Question function:

- asks for examples of insects that attack trees

Structure reasoning:

- this belongs to the pest-and-disease management paragraph

Evidence pattern:

- E lists hemlock wooly adelgid, Asian longhorned beetle, emerald ash borer, and gypsy moth

Takeaway:

- when a question asks for attack examples, outbreak examples, or sick-tree management, go to the salvage / threat paragraph

#### 18 -> B

Question function:

- asks for an alternative name for trees producing low-use wood

Structure reasoning:

- this still belongs to the low-use wood problem paragraph

Evidence pattern:

- the paragraph core idea gets you to B
- the final answer is not in the core sentence itself
- the nearby support sentence gives the exact label: `junk trees`

Takeaway:

- this question proves the method:
- core structure helps locate the paragraph
- support detail provides the final answer

### 6. In fast English scanning, do not give equal attention to every word

When a page is in English, slow readers often feel that attention is "scattered".
This is usually not a pure focus problem.
It is a filtering problem.

What happens:

- too much attention is spent on glue words such as articles, auxiliaries, and low-value connectors
- not enough attention is reserved for the real content words
- as a result, the page feels noisy instead of structured

Better first-pass sequence:

1. classify the page type first
2. read only titles, labels, and other bold text
3. extract content words first, especially nouns and topic words
4. ignore full sentence polish until the second pass
5. choose one interest track instead of trying to process the whole page

For interface-style reading, ask:

- what kind of page is this
- what are the main topic clusters
- which block is useful to me right now
- which blocks can be ignored for now

Worked mini-example:

- `Some Common Mistakes Beginner Quants Make` -> `mistakes`, `beginner quants`
- `Your Agents Produce Slop Because You're Poor` -> `agents`, `slop`
- `Institutional Dataset: Crowdsourced Forward Expectations` -> `institutional dataset`, `forward expectations`

Useful reminder:

`First classify.`
`Then extract content words.`
`Then choose interest.`

### 7. In matching-information style questions, the answer often sits in the contrast, not in the topic words alone

Some paragraphs look as if they are mainly about one topic, but the real answer is hidden in the contrast.

Worked pattern:

- one person appears to have more natural ability
- the group succeeds or matters more
- the paragraph then explains a difference in belief, motivation, or alignment

What this usually means:

- the question is not really about talent
- the key point is the factor that separates the successful group from the missing or weaker case

Useful reminder:

`Topic words show the scene.`
`Contrast reveals the point.`

Mini-example:

- Roy Orbison is described as a greater natural singer
- but the other four understood and believed in Phillips's ambition
- Orbison was not inspired by the goal

So the paragraph is not proving `recognising talent`
and not mainly about `working as a team`
but about `having a shared objective`

### 8. In example-based multiple choice, use the example to locate, but answer the principle

In argumentative passages, a famous person, company, or event often helps you find the right paragraph quickly.
But the final answer is usually not the visible surface topic of that example.

Better sequence:

1. use the proper noun or special example to locate the paragraph
2. read one sentence before and one sentence after the example
3. ask what point the writer is proving through that example
4. match the option to the abstract principle, not the story detail

Common trap:

- the example mentions talent, teamwork, or leadership on the surface
- but the real answer is a deeper idea such as shared goals, commitment, or value fit

Useful reminder:

`Example helps you locate.`
`Principle gives you the answer.`

### 9. In YES/NO/NOT GIVEN, judge the writer's claim, not the topic alone

These questions become difficult when the passage talks about a related area, but not the exact claim in the statement.

Use this check:

1. identify the exact claim in the statement
2. find the sentence where the writer gives a judgement
3. compare the judgement, not just the topic

Decision rule:

- `YES`: same claim
- `NO`: opposite claim
- `NOT GIVEN`: related topic appears, but the exact claim is missing

Worked pattern from the innovation passage:

- `Most people have the potential to be creative` -> `YES`
- `It is easier for smaller companies to be innovative` -> `NOT GIVEN`

Useful reminder:

`Same topic is not enough.`
`You need the same claim.`
